# Getting Support

Thank you for using EKNetwork! Here are several ways to get help:

## üìö Documentation

Before asking a question, please check:

- [README.md](README.md) - main documentation with examples
- [CHANGELOG.md](CHANGELOG.md) - list of changes
- [CONTRIBUTING.md](CONTRIBUTING.md) - contribution guide

## üí¨ Ask a Question

### GitHub Discussions
Use [GitHub Discussions](https://github.com/emvakar/EKNetwork/discussions) for:
- General questions about using the library
- Discussion of ideas and suggestions
- Sharing usage experience

### GitHub Issues
Create an [Issue](https://github.com/emvakar/EKNetwork/issues) for:
- Bug reports
- Feature suggestions
- Questions requiring technical support

**Before creating an Issue:**
1. Check if a similar question has already been asked
2. Use search on existing Issues
3. Provide a minimal code example demonstrating the problem

## üêõ Reporting a Bug

If you found a bug:

1. Check [existing Issues](https://github.com/emvakar/EKNetwork/issues)
2. If not reported, create a new Issue with:
   - Clear description of the problem
   - Steps to reproduce
   - Expected and actual behavior
   - Swift, platform, and EKNetwork version
   - Minimal code example

Use the [Bug Report template](.github/ISSUE_TEMPLATE/bug_report.md).

## üîí Reporting Security Issues

**DO NOT create public Issues for security problems!**

Instead, please send an email to **security@eknetwork.dev** or follow the instructions in [SECURITY.md](SECURITY.md).

## üí° Suggesting a New Feature

We welcome suggestions! Create an Issue with:
- Description of the proposed feature
- Justification of why it's useful
- Usage example

Use the [Feature Request template](.github/ISSUE_TEMPLATE/feature_request.md).

## ü§ù Contributing

If you want to contribute code, please see [CONTRIBUTING.md](CONTRIBUTING.md).

## üìß Email Support
